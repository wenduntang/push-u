# 极简 Agent 编排：50 行实现「代码即编排」，无 JSON 无 DAG

## 背景

做 LLM Agent 时，你肯定遇到过：Tool Calling 的 JSON Schema 难写、解析容易报错、多轮对话状态难维护。有没有更轻量的方案？

## 核心思路：代码即编排

**传统方式**：模型输出 JSON → 解析 → 调用工具 → 再输出 JSON  
**我们的方式**：模型直接输出 Python 代码 → 执行 → 观察结果

逻辑在代码里完成，而不是在 Prompt 里写 if/else。LLM 对 Python 语法的掌握，通常好于对复杂 JSON Schema 的遵循。

## 实现要点

- **RestrictedPython**：编译期拦截 import/open，白名单式开放能力
- **multiprocessing**：超时后真正终止，避免死循环
- **纯函数工具**：`inspect` 自动从 `__doc__` 生成描述，无需手写 Schema
- **Fail-fast**：报错原样回传 LLM，自主 Debug

## 三个 Demo

1. **沙箱运行环境**：Docker/进程隔离，白名单 + 正逆向验证
2. **极简编排引擎**：代码即编排，天气/计算/安全拦截
3. **符号神经融合**：神经(LLM) 生成符号(代码)，执行反馈闭环
